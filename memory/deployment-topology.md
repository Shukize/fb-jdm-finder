JDM Finder runs as three deployed tiers (confirmed live 2026-06-08):

- **Frontend** → GitHub Pages: https://shukize.github.io/fb-jdm-finder/ (static site, repo root, `.nojekyll` present)
- **API** → Render web service named `beater`: https://beater.onrender.com (from `render.yaml` Blueprint, `rootDir: server`)
- **Database** → Neon Postgres (via `DATABASE_URL`, see `server/db.js`)
- **Data source** → Apify Facebook Marketplace actor (`APIFY_TOKEN` / `APIFY_ACTOR`)

`assets/config.js` `apiBase` must match the Render URL (`https://beater.onrender.com`). The Blueprint service name in `render.yaml` was aligned to `beater` so a fresh deploy produces that exact hostname.

Key operational facts:
- `REFRESH_SECRET` is auto-generated by Render — only visible in Render dashboard → beater → Environment. Needed to call `/api/refresh`.
- Render free tier sleeps after ~15 min idle, pausing the in-process scrape cron. Fix: external pinger (e.g. cron-job.org) hitting `/api/refresh?secret=…` every few hours. **Not yet set up.**
- Health: `GET /api/health` returns `db`, `apify`, `liveListings`, `mode` (live|sample), `lastRefresh`.
